## 执行流程

### 第一步：确定时期

1. 检查 $ARGUMENTS 中是否指定了时期：
   - `--era 晚清` 或 `--era 1` → 直接使用
   - 用户输入包含时期关键词（如"晚清""抗战""解放"）→ 自动匹配
   - 未指定时期 → 使用 AskUserQuestion 让用户选择时期

2. 时期匹配规则：
   - "晚清" / "清朝" / "1" → 晚清
   - "民初" / "民国初年" / "2" → 民初
   - "国民革命" / "北伐" / "3" → 国民革命
   - "国共对峙" / "土地革命" / "4" → 国共对峙
   - "抗战" / "抗日战争" / "5" → 抗战
   - "解放战争" / "解放" / "6" → 解放战争
   - "全部" / "all" → 显示全部

### 第二步：读取事件目录

读取 `$CLAUDE_PLUGIN_ROOT/skills/witness/events/catalog.yaml`，获取事件列表。

按用户选择的时期过滤。如果选"全部"，按时期分组显示。

### 第三步：读取完成记录

1. 扫描 `.cc_code/scripts/` 目录下以 `witness_` 开头的文件
2. 在记忆系统中搜索 `witness_` 前缀的记忆条目
3. 交叉验证，标记已完成事件

### 第四步：输出选题列表

按选定时期输出，格式：

```
━━━ 晚清时期 (1839-1911) — 40 个事件 ━━━

  [ ] 虎门销烟 (1839)              视角可选: 销烟现场水勇, 广州行商, ...
  [ ] 鸦片战争 (1840-1842) ▸大事件  视角可选: 清军水勇, 广东渔民, ...
  [✓] 甲午战争 (1894-1895)          视角: 北洋水师水兵  脚本: .cc_code/scripts/witness_jiawu-war_2026-04-16.md
  ...

  已完成: 1/40 | 输入事件名即可生成脚本，如 /cc-code:witness 虎门销烟
```

标注说明：
- `[✓]` 已生成脚本
- `[ ]` 尚未生成
- `▸大事件` 标记 scale 为 large 的事件（适合深度体验）
- `▸中事件` 标记 scale 为 medium 的事件

### 第五步：支持进一步筛选

输出列表后，用户可以：
- `--status done` 只看已完成的
- `--status todo` 只看未完成的
- `--keyword 林则徐` 按关键词过滤（搜索 keywords 字段）
- 切换时期：说"看抗战"或"换时期"

### 第六步：找回脚本

用户说"看看鸦片战争"或点选已完成事件时，读取对应脚本文件，输出完整内容。

## 特殊说明

- 无参数调用时，必须先让用户选时期，再展示列表
- 用户说"全部"时展示所有 156 个事件（按时期分组）
- 列表不要太长，每时期一屏可浏览，必要时分页
